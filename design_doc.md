# 📋 需求完整描述文档

---

## 一、业务背景

用户经营一家网店，商品成本分布在 **15~100 元**之间，采用统一定价公式对所有商品进行标价，并希望通过设置**阶梯满减优惠券**来刺激转化，同时保证每笔订单的成本和利润处于可控范围内。

---

## 二、核心定价模型

### 定价公式
$$标价 = 成本 \times x + y$$

其中：
- **x** = 成本倍数（如 2）
- **y** = 固定加价（如 34，包含 4 元运费 + 30 元固定利润）
- 成本范围：**m ~ n 元**（默认 15~100）

### 利润结构拆解

| 构成项 | 说明 |
|--------|------|
| 成本回收 | `成本 × 1`（保本） |
| 运费覆盖 | 固定 4 元 |
| 固定利润 | 固定 30 元 |
| 浮动利润 | `成本 × (x-1)`，成本越高利润越大 |

---

## 三、阶梯满减优惠券设计需求

### 核心逻辑
- 设置多档 **「满 P 减 Q」** 优惠券
- 每档券自动匹配对应标价区间的商品
- 优惠力度随商品价格提升而递增（高价商品让利更多）
- 所有档位需满足：**优惠后收入 > 成本 + 运费**（绝不亏损）

### 约束条件
```
优惠后实际收入  = 标价 - 优惠金额
利润           = 实际收入 - 成本 - 运费
利润率         = 利润 / 实际收入 × 100%

硬性约束：利润 > 0（任何档位不得亏损）
软性目标：各档利润率尽量均衡，买家折扣感知一致（建议 88~93 折）
```

## 四、后端系统需求

### 技术栈
| 层级 | 技术选型 |
|------|---------|
| Web 框架 | Python Flask |
| 数据库 | SQLite（本地持久化） |
| 图表生成 | Matplotlib + SciPy（服务端渲染） |
| 前后端通信 | RESTful JSON API |

### 数据模型

**配置表 `configs`**
```
id          主键
name        配置名称（如"春季主推款"）
x           成本倍数
y           固定加价
m           成本下限
n           成本上限
created_at  创建时间
```

**优惠券表 `coupons`**
```
id          主键
config_id   外键 → configs.id（级联删除）
tier        档位序号
p           满 p 元触发
q           减 q 元
```

### API 接口清单

**配置 CRUD**
```
GET    /api/configs          查询所有配置列表
POST   /api/configs          新建配置（含优惠券）
GET    /api/configs/:id      查询单个配置详情
PUT    /api/configs/:id      修改配置（含优惠券整体替换）
DELETE /api/configs/:id      删除配置（级联删除优惠券）
```

**优惠券单条操作**
```
POST   /api/configs/:id/coupons   新增一条优惠券
PUT    /api/coupons/:id           修改单条优惠券
DELETE /api/coupons/:id           删除单条优惠券
```

**图表生成**
```
GET    /api/configs/:id/chart     生成并返回 Base64 图片
```

## 六、图表分析需求

点击「确定/生成图表」后，自动生成 **2×2 四张曲线图**：

| 图表 | X 轴 | Y 轴 | 说明 |
|------|------|------|------|
| ① 利润曲线 | 商品成本（元） | 利润（元） | 展示每元成本对应的绝对利润 |
| ② 利润率曲线 | 商品成本（元） | 利润率（%） | 展示利润占实际收入的比例 |
| ③ 用户实际支付曲线 | 商品成本（元） | 实际支付（元） | 优惠后用户实际付款金额 |
| ④ 优惠金额曲线 | 商品成本（元） | 优惠金额（元） | 阶梯状，清晰呈现各档让利 |

### 图表视觉要求
- 成本变化间隔为 **1 元**，曲线平滑（三次样条插值）
- 不同优惠档位用**彩色背景色块**区分
- 每档标注**最大值和最小值**数据标签
- 图表底部显示**公共图例**说明各档位满减规则
- 图表标题显示配置名称、定价公式、成本范围

## 七、数据持久化要求

- 使用 **SQLite** 本地文件存储（`coupons.db`）
- 程序启动时自动建表，无需手动初始化
- 配置删除时**级联删除**所属优惠券（`ON DELETE CASCADE`）
- 支持多套配置独立管理，互不干扰

## 十、核心业务约束总结

```
✅ 任意档位：优惠后利润 > 0（绝不亏本）
✅ 阶梯递增：高价商品享受更大优惠金额
✅ 折扣均衡：各档折扣率控制在 88~93 折区间
✅ 动态适配：修改定价公式后图表自动重新计算

```
